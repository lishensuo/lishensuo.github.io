Elucidating the coordination of RNA processing using short-read and long-read RNA-sequencing methods

使用短读长和长读长 RNA 测序方法阐明 RNA 加工的协调性

  • 时间：06 October 2025
  • 期刊：Nature Reviews Molecular Cell Biology

导读：文章的概念很多，但是没有做过多介绍，所以该文包含了一些背景介绍以及我个人的理解。

0. 背景简单了解：二代 vs 三代 & 新生 RNA 测序#

0.1 测序的三代#

  • 一代（Sanger）

    • 思路：逐条读，很准但通量极低。
    • 现状：现在基本只用于克隆验证、小片段测序。

  • 二代（NGS，短读长）

    • 代表：Illumina。
    • 特点：
      • 读长短（几十～几百 bp）。
      • 准确率高（>99%）。
      • 通量惊人，成本低。
    • 应用：几乎所有传统 RNA‑seq、ChIP‑seq、ATAC‑seq 等 high‑throughput 项目都基于它。

  • 三代（长读长）

    • 代表：PacBio（HiFi）、Oxford Nanopore（ONT）。
    • 特点：
      • 单条 read 很长（几 kb 到几十 kb），可以跨越整条转录本。
      • 初期错误率高、但 PacBio HiFi 等已大幅提高精度。
      • 通量和成本仍不如二代，但在“结构解析”和“单分子层面”有不可替代的优势。
    • 应用：
      • 全长转录本（Iso‑Seq / long‑read RNA‑seq）。
      • 复杂结构变异检测（SV）。
      • 某些方案可直接测 RNA（direct RNA sequencing）。

核心差异总结：

  • 二代：碎片视角，超强定量，适合大 cohort。
  • 三代：整分子视角，可以直接看到一条 RNA 的“头 + 身体 + 尾巴”如何搭配。

0.2 什么是“新生 RNA”？它为什么重要？#

定义：

  • 成熟 mRNA：已经完成加工（剪接、3′ 端切割加尾）、准备翻译或转运的 RNA。
  • 新生 RNA（nascent RNA）：仍然连在线上（Pol II 上）或刚合成不久的 RNA。中间态很多：可能还没剪完、可能还没做 3′ 尾、有的甚至还是带着一堆 intron 的“前体”。

为什么要测新生 RNA？ 常规 bulk RNA‑seq 测到的是稳态丰度（steady‑state），即「合成」+「降解」长期平衡后的结果。很多调控其实发生在：

  • 转录起始 / 延伸速度。
  • 剪接什么时候完成。
  • 终止是否高效。
  • 只看成熟 mRNA，会把这些动力学“压扁”成一个数，难以区分“生产变快”和“降解变慢”。

新生 RNA 测序（nascent RNA‑seq）家族：

  1. 核 run‑on / 转录瞬时抓捕系

    • 代表：GRO‑seq / PRO‑seq / PRO‑cap / NET‑seq 等。
    • 原理：在核内给 Pol II “延伸一小步”并打上标记 → 抓住此时此刻正在合成的 RNA。
    • 能看：Pol II 在基因上的分布（起始/停顿/终止）、某些剪接中间态的比例。

  2. 代谢标记系

    • 代表：4sU/4tU / BrU 标记 + SLAM‑seq, TT‑seq, Bru‑seq 等。
    • 原理：在一定时间窗口投喂可被掺入的新核苷，让“新合成 RNA”带上标签，之后区分“新”与“旧”。
    • 能看：合成速率（谁在快速上调）、剪接速度、mRNA 稳定性/半衰期。

1. 二代短读长：仍然是 RNA‑seq 世界的“量的王者”#

1.1 二代短读长 RNA‑seq 的定位#

数据形态：

  • Read 长度：一般 50–150 bp（单端）或 2×75, 2×150（双端）。
  • 通常测的是 cDNA 文库，最终 mapping 回基因组或转录本注释。

关键特征：

  • 优点：便宜、可扩展（上百/上千样本没压力）、每个位点覆盖度高（统计功效强）、公共数据海量（GTEx, TCGA 等）。
  • 缺点：Read 很短，只能覆盖局部；大部分分析都是**“统计局部事件的频率”**，很难在单分子层面还原完整 isoform 结构；强依赖注释。

结论：它最擅长的是各种“局部加工事件”的定量（谁多谁少、比例差多少），而非“全长结构 + 协同模式”的解析。

1.2 从短读长看 TSS 选择（ATSS）#

  • 问题：同一个基因有多个启动子 / TSS，不同条件下到底偏向用哪一个？
  • 二代数据策略：依赖覆盖在首 exon 区域的 reads 和下游 exon 的 junction reads。
  • 局限：注释 & 模型。如果某个 exon 既可能是某些转录本的首 exon，又是另一些转录本的内部 exon，短读长很难 100% 确定它属于哪种结构。

1.3 从短读长看剪接（AS）#

  • 背景：前体 mRNA 剪接后去掉 intron，连上 exon。某些保留/跳过是可变的（Alternative Splicing, AS）。
  • 二代数据策略：
    • 利用 exon 覆盖度、exon–exon junction reads。
    • 常用指标：PSI（Percent Spliced In）。
    • 工具：rMATS / DEXSeq / MISO 等。
  • 小结：短读长 + 合理模型 = 非常成熟的“大规模 AS 定量管线”。缺点是通常只在“单个事件”层面看，很难知道不同 AS 事件之间在单分子上是怎样组合的。

1.4 从短读长看 APA（alternative polyadenylation）#

  • 背景：同一条 pre‑mRNA 上有多个 poly(A) signal，在不同 PAS 切断导致不同 3′UTR 长度，影响 miRNA 靶点、RBP 结合及翻译效率。
  • 数据来源两大类：
    1. 专门的 3′‑end 测序（如 3′READS, Quant‑seq）：位置精确，画高分辨率 APA 图谱，但看不到上游结构。
    2. 从常规 RNA‑seq 里“抠 APA”：利用 reads 分布梯度推断。优点是不需额外实验、适合大队列；缺点是分辨率有限。

1.5 用短读长 + 新生 RNA 测序看“时间维度”#

常规 RNA-seq 只能看稳态（Steady-state），新生 RNA 测序补上了**“时间/动力学”维度**。

1.5.1 核 run‑on / NET‑seq 家族：直接抓 nascent RNA#

  • 原理：分离细胞核 -> 核内延伸 -> 标记 -> 富集 -> 测序。
  • 能看到：Pol II 空间分布（Pausing, 延伸速率）、剪接中间态（Intron reads 何时减少）。
  • 类比：直接走进生产线车间，用高速相机拍下“半成品”在各工位之间的堆积情况。

1.5.2 代谢标记（4sU / 4tU / BrU 等）+ 二代测序#

  • 原理：投喂特殊核苷 -> 区分新旧 RNA -> 测序。
  • 能看到：转录合成速率、剪接完成时间、降解速率 / 半衰期。

2. 三代长读长：把“结构 + 协调”推到单分子分辨率#

2.1 三代长读长平台的技术特点#

  • PacBio（SMRT / HiFi）：单分子实时测序。HiFi 模式通过循环读同一分子多圈，精度极高（>99.9%）。
  • Oxford Nanopore（ONT）：电流信号判读。读长几乎无限，可做 direct RNA sequencing（捕捉 RNA 修饰）。

2.2 长读长 RNA‑seq 的基本思想#

每条 read 天然携带：

  1. 使用了哪个 TSS / 首 exon。
  2. 中间哪些 exon 被 include / skip。
  3. 内部 splice site 选择如何。
  4. 最后在哪个 PAS 结束。

核心升级：从“局部事件频率”升级到单分子级别的“全程搭配”信息。

2.3 相比二代，三代长读长多看到了哪些维度？#

  1. 完整 isoform 结构：支持直接做 isoform‑level quantification。
  2. 多个事件的“同现/互斥”关系：统计 TSS、AS、PAS 的组合模式。
  3. TSS–AS–APA 三者的联动：回答某个 TSS 是否倾向搭配特定剪接模式和 PAS。
  4. 等位基因特异性 isoform：利用 haplotype 信息区分不同等位基因的 isoform 结构。
  5. 发现新结构：不再严重依赖注释，可发现 novel TSS / exon / PAS / fusion。
  6. 单细胞长读长（scLRS）：在单细胞层面看 isoform 程序。

2.4 二代 vs 三代：总结#

特性 二代短读长 (SRS) 三代长读长 (LRS)
Read 特点 短但多 长但相对少
优势 事件频率统计 + 大样本研究 单分子结构解析 + 事件协同模式分析
核心逻辑 间接反演结构，依赖注释 一条 read 就是“结构证据”
常见组合 大规模“导航地图” 关键样本上的“结构放大镜”

3. 不同 RNA 加工步骤之间的“协调”#

文章的问题升级为：这些加工步骤（TSS 选择、剪接、APA、终止）是“松散的小开关”，还是被组织成几个相对稳定的“RNA 程序”？

3.1 问题：独立开关 vs 协同程序#

传统视角假设各个事件是独立的，总体效果是线性叠加。LRS 数据让我们发问：在真实细胞里，这些“开关”是不是被强行绑定成有限种搭配模式（程序）？

3.2 一些典型的“协调模式”例子#

3.2.1 Promoter–Splicing 协同#

  • 现象：同一基因从不同 TSS 起跑，下游剪接模式也随之切换。
  • 推测机制：染色质状态、TF 结合、Pol II CTD 标记组合、增强子-启动子构象等。

3.2.2 Splicing–APA 协同#

  • 现象：内部某个剪接事件和 3′UTR 长短紧密耦合（例如：Exon A include 总是搭配长 3′UTR）。
  • 推测机制：某个 RBP 同时结合 Intron/Exon 和 3′UTR，充当物理上的“桥”。

3.2.3 Promoter–APA 协同#

  • 现象：不同 TSS 对应不同的 3′UTR 长度偏好。
  • 推测机制：Promoter 级调控可以一路影响到 3′端的选择。

3.3 如何用 LRS 量化“协调程度”#

  1. 定义维度：TSS, Exon, PAS。
  2. 编码向量：把每条 LRS read 编成一个多维标签向量（如 [TSS=1][A=on][PAS=2]）。
  3. 统计频率：看哪些组合是高频模式，哪些被排除。
  4. 计算关联：使用互信息、相关系数等指标。
  5. 跨条件比较：观察“程序”权重的切换。

3.4 从“操作系统配置”的角度看这些协同#

LRS + 协同分析告诉我们：细胞真实在跑的是几套预定义好的“配置档/启动模式”。

  • 配置档 1：TSS1 + A‑in + B‑out + 长 3′UTR
  • 配置档 2：TSS2 + A‑out + B‑in + 短 3′UTR

许多环境变化、发育阶段转换、疾病状态，可能就是在不同配置档之间切换权重，而非“所有小开关逐个独立调”。
